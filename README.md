# admin-lab

`admin-lab` 是一个基于 BoyKing Admin 思路建设的外网实验场项目。

它的核心目标不是直接承载正式业务，而是为内网后台项目提供一个更自由的验证环境，用来提前完成下面这些工作：

- 试验不熟练的新前端组件、新交互方案、新页面结构
- 验证后端新库、新模式、新业务实现方式
- 沉淀可复用的通用模块，例如导入导出、可视化、复杂表单、附件处理、权限联动等
- 将实验通过后的实现整理为可以搬运到内网项目中的参考代码或复用模块

## 项目定位

`admin-lab` 应当长期保持为一个“实验 + 沉淀”型后台项目，而不是新的正式业务系统。

建议遵循以下原则：

- 先验证，再搬运：所有新技术和新模块先在这里试通，再进入内网项目
- 以复用为目标：实验代码尽量抽象成可复制、可裁剪、可独立迁移的模块
- 保持低耦合：避免把实验能力和具体业务流程绑死
- 重视示例价值：每个模块都尽量保留完整示例页、接口和使用说明
- 控制范围：不把它做成通用平台，只围绕后台研发高频需求持续积累

## 推荐建设方向

第一阶段优先沉淀高频能力：

1. 导入导出
2. 图表可视化
3. 通用搜索表单
4. 富文本与附件上传
5. 大表格与复杂列配置
6. 字典、枚举、状态标签统一方案
7. 权限按钮和菜单联动示例

第二阶段沉淀中型业务模块：

1. 通用 CRUD 模板
2. 审批流或状态流页面样例
3. 数据大屏或驾驶舱样例
4. 操作日志、审计、变更记录展示样例
5. 批量操作与异步任务交互样例

## 目录建议

在 BoyKing Admin 脚手架成功拉取后，建议额外补充以下约定：

- `docs/`: 模块说明、迁移说明、踩坑记录
- `server/`: 后端实验模块与通用能力
- `web/`: 前端实验页面与可复用组件
- `examples/` 或 `docs/examples/`: 搬运到内网时的最小接入示例
