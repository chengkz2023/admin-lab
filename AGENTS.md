## Skills

A skill is a set of local instructions stored in a `SKILL.md` file.

### Available skills

- admin-lab-builder: Build, refactor, document, or review `admin-lab` features with the project handbook rules, including module classification (`simulation / component-demo / reusable`), package layout, menu and seed-data updates, and migration-oriented delivery. (file: .codex/skills/admin-lab-builder/SKILL.md)
- boyking-admin-builder: Extend, customize, or bootstrap BoyKing Admin projects while preserving the current slimmed architecture. (file: .codex/skills/boyking-admin-builder/SKILL.md)

### How to use skills

- Trigger by name: if a user mentions `$admin-lab-builder` or the task clearly matches admin-lab handbook-based implementation, use `admin-lab-builder`.
- Trigger by scope: for generic BoyKing scaffold tasks not specific to admin-lab governance, use `boyking-admin-builder`.
- If both apply, use `admin-lab-builder` first for project rules and `boyking-admin-builder` second for scaffold conventions.
- If a named skill path cannot be read, state that briefly and continue with the best fallback.
